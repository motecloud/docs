# Support

Use the right support path for the type of issue.

## Documentation Feedback

Open an issue or pull request in this public docs repository for public documentation corrections, broken links, unclear explanations, or safe example improvements.

## Benchmark Feedback

Use the public benchmarks repository for feedback on published benchmark artifacts, methodology, or reproducibility notes.

## Account Or Tenant Help

Use the hosted product support path for account-specific questions, tenant setup, billing, credentials, or tenant-scoped troubleshooting.

Do not publish tenant identifiers, credentials, screenshots with sensitive data, or support transcripts in public issues.

## Security Reports

Use the private security reporting path described in [SECURITY.md](../../SECURITY.md). Do not open public issues for suspected vulnerabilities.
