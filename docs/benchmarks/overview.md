# Benchmark Overview

MoteCloud publishes public benchmark material to help users understand behavior, methodology, and tradeoffs.

This documentation repository explains benchmark concepts and interpretation. Public benchmark artifacts live in the separate public repository at [github.com/motecloud/benchmarks](https://github.com/motecloud/benchmarks).

## What Belongs Here

This docs repository can include:

- Benchmark overview and goals.
- Methodology explanations.
- Notes about interpreting results.
- Public caveats and limitations.
- Links to published benchmark artifacts.

## What Belongs In The Benchmarks Repo

The benchmarks repository can include:

- Published result files.
- Public datasets or dataset manifests.
- Reproduction notes.
- Public comparison tables.
- Methodology snapshots.

## What Stays Private

Internal tuning notes, unpublished research, private artifact paths, tenant-specific evaluation data, proprietary thresholds, and unreleased benchmark datasets should not be published.

## Interpreting Results

Benchmarks are evidence, not a complete product story. Results should explain what was measured, what was not measured, what baseline was used, and how the public reader should interpret the numbers.
