# Benchmark Methodology

Public benchmark methodology should help readers understand what was measured and how results were produced.

## Required Elements

Each public benchmark should describe:

- The benchmark goal.
- The task format.
- The dataset or dataset manifest.
- The evaluated systems or baselines.
- The scoring metric.
- The run date or version.
- Known limitations.

## Reproducibility

Where possible, public benchmark artifacts should include enough information for a reader to reproduce or audit the result without accessing private infrastructure or restricted datasets.

## Metric Clarity

Metrics should be named precisely. Retrieval metrics, reader accuracy, latency, coverage, and user-workflow measures answer different questions and should not be blended without explanation.

## Publication Boundary

Do not publish internal experiment tuning, private artifact paths, tenant data, unreleased datasets, proprietary thresholds, or source links to private repositories.
