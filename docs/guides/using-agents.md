# Using Agents

Agents work best with MoteCloud when memory is part of the workflow rather than an afterthought.

## Before Work Starts

Prepare context for the specific task. The task description should be clear enough to retrieve relevant memories, conventions, and prior decisions.

## During Work

Append observations after meaningful events:

- A useful discovery.
- A decision that affects the path forward.
- A validation result.
- An error with a reusable lesson.
- A contradiction or uncertainty that should be visible later.

Keep observations concise. A future agent should be able to scan them quickly.

## After Work Finishes

Close the session with a short summary. Persist durable findings only when they are validated and likely to help future work.

## Good Agent Behavior

Good agents use memory to become more consistent without becoming careless. They retrieve context, check scope, validate important claims, and avoid storing sensitive or speculative information.

## Public Examples

Public examples should use placeholder values and safe scenarios. They should never include real tenant data, production credentials, private repository links, internal operational commands, or admin-only procedures.
