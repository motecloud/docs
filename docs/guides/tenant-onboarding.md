# Tenant Onboarding

Tenant onboarding is the path from public evaluation to authenticated use.

This public page explains the general shape of onboarding. Account-specific setup, tenant identifiers, credentials, billing details, and tenant-scoped runbooks belong in hosted tenant documentation after sign-in.

## Public Evaluation

Before creating a tenant, review:

- [Quickstart](../quickstart.md)
- [Context priming](../concepts/context-priming.md)
- [Using MoteCloud with MCP](using-mcp.md)
- [Benchmark overview](../benchmarks/overview.md)

## Authenticated Setup

After sign-in, tenant users should use the hosted tenant docs for setup steps that depend on account state, credentials, or tenant permissions.

## Roles And Access

Tenant access should follow least privilege. Users should only receive the permissions needed for their work.

## What Not To Publish

Do not publish tenant identifiers, API credentials, account screenshots with sensitive details, billing information, support transcripts, or tenant-specific configuration in public issues or documentation.
