# Security Policy

MoteCloud takes security reports seriously.

## Reporting A Vulnerability

Please do not open a public issue for suspected vulnerabilities. Use the security contact published on the MoteCloud website, or contact the maintainers through the private reporting channel provided there.

When reporting, include:

- A concise description of the issue.
- The affected public surface, if known.
- Reproduction steps that do not include secrets or tenant data.
- Any relevant timestamps or request identifiers that are safe to share privately.

## Public Documentation Boundary

This repository may describe public security posture and responsible disclosure process. It does not publish internal security evidence, admin runbooks, incident response procedures, private infrastructure details, credentials, secrets, tenant-specific findings, or exploit instructions.

## Safe Handling

Do not include API keys, access tokens, JWTs, private keys, session cookies, production tenant identifiers, customer data, or private repository links in issues, pull requests, or discussions.
